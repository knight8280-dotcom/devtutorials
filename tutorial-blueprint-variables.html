Understanding Variables

Variables are containers that store information in your game. Think of them like labeled boxes where you can put data and retrieve it later. Every variable has a name, a type (what kind of data it holds), and a value (the actual data).

Variable Types Reference

Boolean (True/False)

Use for: On/off states, yes/no conditions, flags

Examples:

  • bIsDead - Is the character dead?
  • bDoorIsLocked - Is the door locked?
  • bHasKey - Does player have the key?
  • bIsInCombat - Is player fighting?
Naming Convention: Boolean variables start with "b" (e.g., bIsOpen, not IsOpen)

Integer (Whole Numbers)

Use for: Counts, IDs, whole number values

Range: -2,147,483,648 to 2,147,483,647

Examples:

  • AmmoCount - Number of bullets (0, 1, 2, 3...)
  • PlayerLevel - Character level (1, 2, 3...)
  • EnemyCount - How many enemies alive
  • Score - Player score (1000, 2500, etc.)
Warning: Integers can't store decimals. 10.5 becomes 10. Use Float for decimals.

Float (Decimal Numbers)

Use for: Measurements, percentages, any decimal value

Precision: ~7 decimal places

Examples:

  • Health - 100.0, 75.5, 32.8
  • Speed - 600.0, 450.25
  • DamageMultiplier - 1.5, 2.0, 0.75
  • CooldownTimer - 3.5 seconds, 1.2 seconds
Best Practice: Use Floats for anything that needs decimals - health, speed, timers, distances.

String (Text)

Use for: Names, messages, any text

Examples:

  • PlayerName - "John", "Sarah", "Hero123"
  • QuestDescription - "Find the lost sword"
  • DialogueText - "Hello, traveler!"
  • ItemName - "Health Potion", "Iron Sword"
Performance: Strings use more memory than numbers. Don't use for data that could be numbers/booleans.

Vector (3D Coordinates)

Use for: Positions, directions, velocities in 3D space

Components: X (forward/back), Y (left/right), Z (up/down)

Examples:

  • SpawnLocation - (1000, 500, 200)
  • MoveDirection - (1, 0, 0) = forward
  • PlayerVelocity - Current movement speed in 3D
  • TargetPosition - Where to move to

Rotator (Rotation)

Use for: Rotation angles

Components: Pitch (up/down), Yaw (left/right), Roll (tilt)

Examples:

  • PlayerRotation - Which way character faces
  • CameraRotation - Camera angle
  • DoorRotation - Open/closed angle

Step-by-Step: Creating Your First Variable

Exercise 1: Player Health Variable

Let's create a health system from scratch.

Step 1: Create Blueprint

  1. Content Browser → Right-click → Blueprint Class → Actor
  2. Name it "BP_HealthTest"
  3. Double-click to open

Step 2: Add Health Variable

  1. In My Blueprint panel, click + Variable
  2. Name it "Health"
  3. Click the variable to select it
  4. In Details panel:
    • Variable Type: Float (click dropdown)
    • Category: "Stats" (organize your variables)
    • Instance Editable: ✓ Check (can edit per instance)
    • Default Value: 100.0
  5. Click Compile to save

Step 3: Add Max Health Variable

  1. Create another variable: "MaxHealth"
  2. Type: Float
  3. Category: "Stats"
  4. Instance Editable: ✓ Check
  5. Default Value: 100.0
  6. Compile

Step 4: Display Health on Screen

  1. Go to Event Graph
  2. Find "Event BeginPlay" node
  3. Drag from execution pin → search "Print String"
  4. Drag Health variable into graph (hold Ctrl for GET)
  5. Drag from Health pin → "Append" node
  6. Set Append A: "Current Health: "
  7. Connect Append output to Print String "In String"

Step 5: Test It

  1. Click Compile and Save
  2. Drag BP_HealthTest into your level
  3. Select it → Details panel shows Health and MaxHealth
  4. Try changing Health to 75
  5. Press Play
  6. See "Current Health: 75.0" on screen!

Variable Scope & Access

Instance Editable

When checked, you can set different values for each copy of the Blueprint.

Example: 3 enemies with different health values:

  • Enemy1: Health = 100
  • Enemy2: Health = 150
  • Enemy3: Health = 50

Use when: Each instance needs unique values

Expose on Spawn

When checked, you must provide a value when creating the Blueprint via code.

Use when: Value is required and shouldn't use default

Blueprint Read Only

Can read the value but not change it in Blueprints (only C++ can change).

Use when: Displaying data that shouldn't be modified

Practical Examples

Example 1: Ammo System

Variables needed:

  • CurrentAmmo (Integer) - Bullets in magazine
  • MaxAmmo (Integer) - Magazine capacity
  • ReserveAmmo (Integer) - Extra bullets

Logic:

When player shoots:
  - CurrentAmmo = CurrentAmmo - 1
  - If CurrentAmmo <= 0:
    - Can't shoot (play click sound)

When player reloads:
  - If ReserveAmmo > 0:
    - BulletsToReload = MaxAmmo - CurrentAmmo
    - If ReserveAmmo >= BulletsToReload:
      - CurrentAmmo = MaxAmmo
      - ReserveAmmo = ReserveAmmo - BulletsToReload
    - Else:
      - CurrentAmmo = CurrentAmmo + ReserveAmmo
      - ReserveAmmo = 0

Example 2: Door System

Variables needed:

  • bIsOpen (Boolean) - Door state
  • bIsLocked (Boolean) - Can it be opened?
  • RequiredKeyName (String) - Which key needed
  • OpenRotation (Rotator) - Door open angle
  • ClosedRotation (Rotator) - Door closed angle
  • OpenSpeed (Float) - How fast it opens

Common Mistakes

1. Using Float for Counts

Wrong: EnemyCount (Float) = 5.0

Right: EnemyCount (Integer) = 5

Why: You can't have 5.5 enemies. Use Integer for countable things.

2. Not Using Default Values

Problem: Variables start at 0 or empty by default

Solution: Always set sensible defaults (Health = 100, not 0)

3. Poor Naming

Bad: x, temp, var1

Good: PlayerHealth, MaxSpeed, JumpHeight

Why: Clear names make code readable and maintainable

Quick Reference: Choosing Variable Types

If you need to store... Use this type Example
Yes/No, True/False Boolean bIsAlive, bHasKey
Whole numbers, counts Integer AmmoCount, Score
Decimals, measurements Float Health, Speed, Time
Text, names, messages String PlayerName, ItemName
3D position, direction Vector SpawnPoint, Velocity
Rotation angles Rotator CameraRotation
Reference to object Object Reference TargetEnemy, DoorRef
