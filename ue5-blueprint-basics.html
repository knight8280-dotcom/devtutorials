Detailed Chapter Breakdown

Chapter 1: Your First Blueprint

1.1 Creating a Blueprint Class

  1. Open Unreal Engine 5.6 and create/open your project
  2. In Content Browser, right-click → Blueprint Class
  3. Select Actor as parent class
  4. Name it "BP_RotatingCube"
  5. Double-click to open Blueprint Editor

1.2 Understanding the Blueprint Interface

  • Viewport: Visual preview of your Blueprint
  • Components Panel: Add/manage components
  • My Blueprint Panel: Variables, functions, graphs
  • Details Panel: Properties of selected item
  • Event Graph: Where logic happens

1.3 Adding a Component

  1. Click "Add" in Components panel
  2. Select "Static Mesh"
  3. In Details, set Static Mesh to "Cube" (from Engine Content)
  4. The cube appears in viewport

Chapter 2: Making It Move

2.1 Event Tick Basics

  1. Go to Event Graph tab
  2. You'll see "Event BeginPlay" and "Event Tick"
  3. Event Tick runs every frame (~60 times per second)

2.2 Rotate the Cube

  1. From Event Tick's execution pin, drag out
  2. Search for "Add Actor Local Rotation"
  3. Connect Delta Seconds (from Event Tick) to a Multiply node
  4. Set multiplier to 50.0
  5. Connect result to Z (Yaw) rotation input
  6. This rotates at 50 degrees per second
Pro Tip: Always multiply rotations by Delta Seconds to ensure frame-rate independent movement!

2.3 Test Your Blueprint

  1. Click Compile (checkmark icon)
  2. Click Save
  3. Drag BP_RotatingCube from Content Browser into your level
  4. Press Play - your cube rotates!

Chapter 3: Variables & User Input

3.1 Creating Variables

  1. In My Blueprint panel, click "+ Variable"
  2. Name it "RotationSpeed"
  3. Set type to Float
  4. In Details, check "Instance Editable" and "Expose on Spawn"
  5. Set Default Value to 50.0
  6. Click Compile to confirm

3.2 Using Variables

  1. In Event Graph, drag RotationSpeed variable in
  2. Choose GET (blue node)
  3. Replace the hardcoded 50.0 with this variable
  4. Now you can change rotation speed per-instance in the level!

3.3 Variable Types Reference

  • Boolean: True/False (checkboxes, toggles)
  • Integer: Whole numbers (score, ammo count)
  • Float: Decimal numbers (speed, health, damage)
  • String: Text (player name, dialogue)
  • Vector: 3D coordinates (location, direction)
  • Rotator: Rotation values (pitch, yaw, roll)
  • Transform: Location + Rotation + Scale combined
  • Object Reference: Reference to another actor/component

Chapter 4: Functions for Reusable Logic

4.1 Creating a Custom Function

  1. In My Blueprint panel, click "+ Function"
  2. Name it "DealDamage"
  3. Opens the function graph

4.2 Adding Inputs and Outputs

  1. Select the function entry node
  2. In Details, under Inputs, click "+"
  3. Add input: "DamageAmount" (Float)
  4. Add input: "DamageType" (String)
  5. Add output: "WasFatal" (Boolean)

4.3 Function Logic Example

  1. Get player's current Health variable
  2. Subtract DamageAmount from Health
  3. Set Health to new value
  4. Branch: If Health <= 0, return WasFatal = true, else false
  5. Connect to Return Node

4.4 Calling Your Function

  1. In Event Graph, right-click and search for "DealDamage"
  2. Your custom function appears!
  3. Connect it after any event
  4. Set DamageAmount and read WasFatal output

Chapter 5: Flow Control

5.1 Branch (If/Else)

Example: Check if player has enough mana
1. Add node: "Branch"
2. Condition: CurrentMana >= SpellCost
3. True pin: Cast spell, subtract mana
4. False pin: Show "Not enough mana" message

5.2 Sequence (Do Multiple Things)

Example: Player dies
1. Add node: "Sequence"
2. Then 0: Play death animation
3. Then 1: Drop all items
4. Then 2: Ragdoll physics
5. Then 3: Show respawn UI

5.3 For Loop (Repeat Actions)

Example: Damage all enemies in range
1. Add node: "For Loop"
2. First Index: 0
3. Last Index: Length of EnemyArray - 1
4. Loop Body: Call DealDamage on Array[Index]

5.4 Switch on String/Int

Example: Different behavior per weapon type
1. Add node: "Switch on String"
2. Add pin for each weapon: "Sword", "Bow", "Staff"
3. Connect different logic to each pin

Chapter 6: Blueprint Communication

6.1 Direct Reference (Fastest)

  1. Create variable of type "Actor" or specific class
  2. Drag reference into level, select the actor you want
  3. In Blueprint, GET the reference and call functions on it
  4. Use when: You know exactly which actor at design time

6.2 Cast To (Type Checking)

  1. When you have a generic Actor reference
  2. Use "Cast to BP_PlayerCharacter" node
  3. Success pin: It was that type, access its functions
  4. Fail pin: It wasn't, do something else

6.3 Event Dispatchers (Broadcasting)

  1. Create Event Dispatcher: "OnHealthChanged"
  2. When health changes, call "OnHealthChanged" dispatcher
  3. Other Blueprints bind to this event
  4. Use when: Multiple things need to react to one event

6.4 Blueprint Interfaces (Clean Communication)

  1. Create Blueprint Interface: "BPI_Damageable"
  2. Add function: "TakeDamage"
  3. Implement interface on any Blueprint that can take damage
  4. Call interface without casting - it just works!

Chapter 7: Practical Example - Door System

Complete Interactive Door Blueprint

Components:

  • Static Mesh: Door frame and door
  • Box Collision: Trigger for "Press E to Open"

Variables:

  • bIsOpen (Boolean) - Is door currently open?
  • bIsLocked (Boolean) - Is door locked?
  • RequiredKey (String) - Key name needed to unlock
  • OpenRotation (Rotator) - Door's open position
  • ClosedRotation (Rotator) - Door's closed position
  • OpenSpeed (Float) - How fast door moves

Logic:

  1. Event BeginPlay:
    • Store initial rotation as ClosedRotation
    • Calculate OpenRotation (add 90° to Yaw)
  2. On Player Presses E:
    • Branch: Is bIsLocked?
    • True: Check if player has RequiredKey
      • Yes: Set bIsLocked = false, proceed to open
      • No: Display "Locked - need [KeyName]"
    • False: Toggle bIsOpen
    • Start Timeline for smooth rotation
  3. Timeline (0 to 1 over 1 second):
    • Lerp between ClosedRotation and OpenRotation
    • Set door's rotation to lerped value
    • Play door sound effect

Chapter 8: Debugging Your Blueprints

8.1 Print String

  • Add "Print String" node anywhere
  • Shows text on screen during play
  • Perfect for checking variable values

8.2 Breakpoints

  • Right-click any node → "Add Breakpoint"
  • Red dot appears
  • During play, execution pauses at breakpoint
  • Inspect all variable values at that moment
  • Step through execution one node at a time

8.3 Watch Values

  • Right-click variable → "Watch This Value"
  • Shows live value during gameplay
  • No need to print
