Introduction to Character Creation in UE5.6

This comprehensive guide covers the complete character creation pipeline in Unreal Engine 5.6. You'll learn how to import characters, set up skeletal meshes, create animations, build character controllers, and implement advanced features like IK (Inverse Kinematics), blend spaces, and AI behaviors.

Prerequisites: Basic UE5 knowledge, understanding of Blueprints recommended. For 3D modeling sections: Blender, Maya, or similar 3D software (optional).

Part 1: Getting a Character Model

Method 1: Using MetaHuman (Easiest)

1.1 Access MetaHuman Creator

  1. Open Epic Games Launcher
  2. Go to Library tab
  3. Find and launch "MetaHuman" (or download if needed)
  4. MetaHuman Creator opens in your browser
  5. Sign in with your Epic account

1.2 Create Your MetaHuman

  1. Click "Create MetaHuman"
  2. Choose a preset or start from scratch
  3. Customize features:
    • Face: Eyes, nose, mouth, ears, jaw shape
    • Body: Height, build, proportions
    • Hair: Style, color, facial hair
    • Clothing: Outfits, accessories
    • Skin: Tone, texture, makeup
  4. Click "Download" when finished
  5. Select your UE5.6 project
  6. MetaHuman appears in Content/MetaHumans folder
Why MetaHuman? Photorealistic humans with full facial rig, body animations, and LODs. Production-ready in minutes!

Method 2: Import Custom Character

2.1 Prepare Your Character Model

If you have a character from Blender, Maya, or other 3D software:

  • Format: Export as FBX or USD
  • Scale: 1 unit = 1cm in UE5 (character should be ~180cm tall)
  • Position: Character at world origin (0,0,0)
  • Rotation: Facing +Y axis (forward)
  • Pose: T-pose or A-pose for rigging
  • Skeleton: Must be rigged with bones/armature

2.2 Import FBX into UE5.6

  1. In UE5 Content Browser, navigate to Content/Characters folder (create if needed)
  2. Click "Import" button
  3. Select your FBX file
  4. In the FBX Import Options dialog:
    • Mesh:
      • Skeletal Mesh: Enabled
      • Import Mesh: Enabled
      • Import Materials: Enabled
      • Import Textures: Enabled
    • Skeleton:
      • If first import: "None" (creates new skeleton)
      • If you have existing skeleton: Select it
    • Transform:
      • Import Uniform Scale: 1.0
  5. Click "Import All"

This creates:

  • Skeletal Mesh asset (SK_YourCharacter)
  • Skeleton asset (SK_YourCharacter_Skeleton)
  • Physics Asset (SK_YourCharacter_PhysicsAsset)
  • Materials and Textures

Part 2: Setting Up Character Movement

Step 4: Create Character Blueprint

4.1 Create Blueprint Class

  1. In Content Browser, right-click in Content/Characters folder
  2. Select Blueprint Class
  3. Choose "Character" as parent class
  4. Name it "BP_PlayerCharacter"
  5. Double-click to open Blueprint Editor

4.2 Assign Skeletal Mesh

  1. In the Blueprint Editor, select "Mesh" component (in Components panel)
  2. In Details panel:
    • Skeletal Mesh: Select your imported character mesh
    • Location: Z = -90 (adjust so feet are at capsule bottom)
    • Rotation: Z = -90 (so character faces forward)
  3. The character should now appear in the viewport facing the right direction
Tip: The CapsuleComponent is the collision shape. The mesh is just visual. Make sure the capsule fits around your character!

Step 5: Configure Character Movement

5.1 Character Movement Component Settings

  1. Select "CharacterMovement" component
  2. In Details panel, configure:
    • Max Walk Speed: 600 (default running speed)
    • Max Walk Speed Crouched: 300
    • Jump Z Velocity: 600 (jump height)
    • Air Control: 0.2 (movement control while airborne)
    • Ground Friction: 8.0
    • Braking Deceleration Walking: 2048

5.2 Add Movement Input

  1. Go to Edit ? Project Settings ? Input
  2. Under "Bindings", add:
    • Action Mappings:
      • Jump - Space Bar
      • Sprint - Left Shift
    • Axis Mappings:
      • MoveForward - W (Scale: 1.0), S (Scale: -1.0)
      • MoveRight - D (Scale: 1.0), A (Scale: -1.0)
      • Turn - Mouse X (Scale: 1.0)
      • LookUp - Mouse Y (Scale: -1.0)

5.3 Implement Movement in Blueprint

In your BP_PlayerCharacter, go to Event Graph:

Forward/Backward Movement:

  1. Right-click ? search "MoveForward"
  2. Select "InputAxis MoveForward" event
  3. From the execution pin, add "Add Movement Input" node
  4. Connect:
    • World Direction: Get Control Rotation ? Get Forward Vector
    • Scale Value: Connect from MoveForward's Axis Value

Left/Right Movement:

  1. Add "InputAxis MoveRight" event
  2. Add "Add Movement Input" node
  3. Connect:
    • World Direction: Get Control Rotation ? Get Right Vector
    • Scale Value: Connect from MoveRight's Axis Value

Camera Rotation:

  1. Add "InputAxis Turn" event
  2. Add "Add Controller Yaw Input" node
  3. Connect Turn's Axis Value to the input
  4. Add "InputAxis LookUp" event
  5. Add "Add Controller Pitch Input" node
  6. Connect LookUp's Axis Value to the input

Jumping:

  1. Add "InputAction Jump" event
  2. From Pressed: Add "Jump" node
  3. From Released: Add "Stop Jumping" node
Quick Setup: If using Third Person template, these inputs are already configured! Just assign your mesh to the template character.

Step 6: Camera Setup

6.1 Add Camera Components

  1. In Components panel, click "Add"
  2. Add "Spring Arm" component (attach to CapsuleComponent)
  3. Add "Camera" component (attach to SpringArm)

6.2 Configure Spring Arm

  1. Select SpringArm component
  2. In Details:
    • Target Arm Length: 300 (camera distance)
    • Socket Offset: Z = 60 (camera height)
    • Enable Camera Lag: Checked (smooth camera)
    • Camera Lag Speed: 10.0
    • Use Pawn Control Rotation: Checked

6.3 Character Rotation Settings

  1. Select main Character component (BP_PlayerCharacter)
  2. In Details, under "Pawn":
    • Use Controller Rotation Yaw: Unchecked
    • Use Controller Rotation Pitch: Unchecked
    • Use Controller Rotation Roll: Unchecked
  3. Select CharacterMovement component
  4. Under "Character Movement (Rotation Settings)":
    • Orient Rotation to Movement: Checked
    • Rotation Rate: Z = 540.0

Part 3: Animation Setup

Step 7: Import Animations

7.1 Get Animations from Mixamo

  1. Go to mixamo.com
  2. Select your character (or use default)
  3. Click "Animations" tab
  4. Search and download these essential animations:
    • Idle: "Breathing Idle" or "Standing Idle"
    • Walk: "Walking" or "Walking Forward"
    • Run: "Fast Run" or "Running"
    • Jump Start: "Jump"
    • Jump Loop: "Falling Idle"
    • Jump Land: "Landing"
  5. Download settings:
    • Format: FBX
    • Skin: Without Skin (unless it's your character)
    • Frame Rate: 30fps
    • In Place: Checked (for idle/combat), Unchecked (for locomotion)

7.2 Import Animation FBX Files

  1. In UE5 Content Browser, create folder: Content/Characters/Animations
  2. Click "Import"
  3. Select all downloaded animation FBX files
  4. In Import Options:
    • Skeleton: Select your character's skeleton (IMPORTANT!)
    • Import Mesh: Unchecked (we only want animation)
    • Animation Length: Exported Time
  5. Click "Import All"
Critical: All animations MUST use the same Skeleton asset as your character mesh! Mismatched skeletons won't work.

Step 8: Create Animation Blueprint

8.1 Create Animation Blueprint Asset

  1. Right-click in Content/Characters/Animations folder
  2. Select "Animation ? Animation Blueprint"
  3. Choose your character's Skeleton
  4. Choose UAnimInstance as parent class
  5. Name it "ABP_PlayerCharacter"
  6. Double-click to open

8.2 Create Variables

In the Animation Blueprint, go to "My Blueprint" panel and add these variables:

  • Speed (Float) - character movement speed
  • Direction (Float) - movement direction
  • IsInAir (Boolean) - is character falling/jumping
  • IsCrouching (Boolean) - is character crouched

8.3 EventGraph - Update Variables

  1. Go to "EventGraph" tab
  2. From "Event Blueprint Update Animation" event:
    • Add "Try Get Pawn Owner" node
    • Cast to your character (BP_PlayerCharacter)
    • From character reference:
      • Get Velocity ? Vector Length ? Set Speed
      • Get Movement Component ? Is Falling ? Set IsInAir
      • Get Movement Component ? Is Crouching ? Set IsCrouching

Step 9: Animation State Machine

9.1 Create State Machine

  1. Go to "AnimGraph" tab
  2. Right-click ? "Add New State Machine"
  3. Name it "Locomotion"
  4. Connect to "Output Pose"
  5. Double-click the State Machine to open

9.2 Create States

Right-click in the State Machine graph and add these states:

  1. Idle/Walk/Run - locomotion blend space (we'll create next)
  2. Jump_Start - jump takeoff animation
  3. Jump_Loop - in-air falling animation
  4. Jump_Land - landing animation

9.3 Create Transitions

Connect states with transitions (drag from one state to another):

  • Entry ? Idle/Walk/Run (default state)
  • Idle/Walk/Run ? Jump_Start
    • Condition: IsInAir == true
  • Jump_Start ? Jump_Loop
    • Automatic (check "Automatic Rule Based on Sequence Player")
  • Jump_Loop ? Jump_Land
    • Condition: IsInAir == false
  • Jump_Land ? Idle/Walk/Run
    • Automatic

Step 10: Create Blend Space (Idle/Walk/Run)

10.1 Create Blend Space 1D

  1. In Content/Characters/Animations, right-click
  2. Select "Animation ? Blend Space 1D"
  3. Choose your Skeleton
  4. Name it "BS_Locomotion"
  5. Double-click to open

10.2 Configure Blend Space

  1. In the Details panel (Axis Settings):
    • Name: Speed
    • Minimum Axis Value: 0.0
    • Maximum Axis Value: 600.0 (your max walk speed)
  2. Drag animations from Asset Browser to the blend space grid:
    • Idle animation at position 0.0
    • Walk animation at position 300.0
    • Run animation at position 600.0
  3. Press play button to preview - drag the slider to see smooth blending!
  4. Save

10.3 Use Blend Space in State Machine

  1. Go back to your Animation Blueprint ? AnimGraph ? State Machine
  2. Double-click the "Idle/Walk/Run" state
  3. Drag in your BS_Locomotion blend space
  4. Connect Speed variable to the Speed input
  5. Connect output to Result

10.4 Configure Jump States

For each jump state, double-click and add the animation:

  1. Jump_Start state: Add "Play Jump" animation node
  2. Jump_Loop state: Add "Play Falling" animation node (set to loop)
  3. Jump_Land state: Add "Play Landing" animation node

Step 11: Connect Animation Blueprint to Character

  1. Open your BP_PlayerCharacter blueprint
  2. Select Mesh component
  3. In Details panel, find "Animation" section
  4. Set "Anim Class" to your ABP_PlayerCharacter
  5. Compile and Save

Part 4: Advanced Character Features

Step 12: IK (Inverse Kinematics) for Feet

12.1 Add IK Rig

  1. Right-click on your Skeleton asset
  2. Select "Create ? IK Rig"
  3. Name it "IK_Character"
  4. Open the IK Rig
  5. Add IK goals:
    • Left Foot IK Goal (foot_l bone)
    • Right Foot IK Goal (foot_r bone)
  6. Save

12.2 Add IK in Animation Blueprint

  1. In your Animation Blueprint AnimGraph
  2. After your State Machine output, add "Two Bone IK" nodes
  3. Configure one for each foot
  4. Use Line Trace to ground to get IK target locations
Result: Characters' feet will properly plant on uneven terrain!

Step 13: Add Advanced Movements

13.1 Sprinting

  1. In BP_PlayerCharacter, add variable: bIsSprinting (Boolean)
  2. Add Sprint input event (Left Shift)
  3. On Pressed: Set bIsSprinting = true, Set Max Walk Speed = 1000
  4. On Released: Set bIsSprinting = false, Set Max Walk Speed = 600

13.2 Crouching

  1. Add Crouch input (C key)
  2. On Pressed: Call "Crouch" function
  3. On Released: Call "UnCrouch" function
  4. Add crouch animations to your blend space

13.3 Dodge/Roll

  1. Import dodge/roll animation from Mixamo
  2. Create Animation Montage from the animation
  3. In character blueprint, add input for dodge (Space while moving)
  4. Play montage: "Play Anim Montage" node
  5. During montage, add forward velocity with "Launch Character"

Part 5: AI Character Setup

Step 14: Create AI Character

14.1 Create AI Controller

  1. Right-click ? Blueprint Class ? AIController
  2. Name it "AIC_Enemy"

14.2 Create AI Character Blueprint

  1. Create new Character blueprint: "BP_EnemyCharacter"
  2. Assign mesh and animations
  3. In Details, under "Pawn":
    • AI Controller Class: AIC_Enemy
    • Auto Possess AI: Placed in World or Spawned

Step 15: Behavior Tree

15.1 Create Behavior Tree Asset

  1. Right-click ? Artificial Intelligence ? Behavior Tree
  2. Name it "BT_Enemy"
  3. Create associated Blackboard (BB_Enemy)

15.2 Set Up Blackboard

  1. Open BB_Enemy
  2. Add keys:
    • TargetActor (Object type: Actor) - player to chase
    • PatrolLocation (Vector) - where to patrol
    • CanSeePlayer (Boolean)

15.3 Basic Behavior Tree

  1. Open BT_Enemy
  2. Add Selector node from Root
  3. Add sequences:
    • Sequence 1: Chase Player
      • Decorator: Blackboard Based Condition ? CanSeePlayer == true
      • Task: Move To (Target: TargetActor)
    • Sequence 2: Patrol
      • Service: Find Random Location
      • Task: Move To (Target: PatrolLocation)
      • Task: Wait (random 2-5 seconds)

15.4 Run Behavior Tree

  1. In AIC_Enemy, Event Graph
  2. On Begin Play: "Run Behavior Tree" ? Select BT_Enemy

Part 6: Character Stats & Health

Step 16: Health System

16.1 Add Health Variables

In BP_PlayerCharacter, add variables:

  • Health (Float) - Default: 100.0
  • MaxHealth (Float) - Default: 100.0
  • IsDead (Boolean) - Default: false

16.2 Create TakeDamage Function

  1. In Functions panel, add new function: "TakeDamage"
  2. Add input: DamageAmount (Float)
  3. Logic:
    • Health = Health - DamageAmount
    • If Health <= 0: Call Die function
    • Else: Play damage reaction animation

16.3 Create Die Function

  1. Add function: "Die"
  2. Set IsDead = true
  3. Disable input
  4. Play death animation montage
  5. Enable ragdoll physics (Set Simulate Physics)
  6. Optionally: Respawn after delay
